WESC Information Model

The Water and Energy Supply and Consumption (WESC) data standard has been developed by CSIRO to support a common represention of data for supply and consumption of water and energy and its delivery via standardised OGC services. The information model allows for data to be delivered in a consistent format and is the basis for the exchange format (XML schemas) for encoding datasets. Aggregated consumption information model

Simple tabular view of consumption aggregated by some spatialRepresentation (e.g. locality, zone substation region).

This accomodates the situation where it is desirable to have an aggregated consumption reported for a region, i.e. a set of meter readings for properties in a given region such as a metropolitan region or suburb. Both geometryID and spatialRepresentationprovides the fields for denoting the region identifier, or the concrete shape/polygon for the region.

Consumption information model

This provides a fine-grained demand-centric view of resource consumption based on meter readings for a meter located on a property. This could be a single-dwelling, in which case, there is a 1-to-1 mapping between the meter and the property IDs, as well as other property types where there is a 1-to-many mapping between the property and the meter.

The top-level class is MeterReading and has the following subclasses:WaterMeterReading, GasReading, and ElectricityMeter Reading.

This model accomodates encoding of data for various types of readings, such as manual meter readings, calculated via the resultMethod vocabulary.

Supply information model

The WESC Supply information model is intended to standardise supply related data (water, electricity, zone substation electricity). UtilitySupplyData provides the parent class to allow inheritance of common supply data properties and extension points for new types. These include properties to capture the name and type of supply region (i.e. local government area, jurisdiction, statistical area), the name of the supplying utility and its identifier, the total amount of water or electricity supplied along with the supply units of measure and a link to the associated vocabulary.

A vocabulary to enumerate the time period property (timeName) was provided, to avoid calculating it from the startDeliveryTime and endDeliveryTime properties. A general geometry data type for the supply region allows representing the supply region as multi-part polygons, single polygons or points, depending on scale and use case requirements.

GML SF-1 WESCML information model

The WESCML-SF1 information model provides a detailed extension for water and electricity supply and consumption data to the Open Geospatial Consortium (OGC) Observations and Measurements version 2.0 (O&M). O&M provides a conceptual model, with an associated Geography Markup Language version 3.2 (GML) implementation in XML, for describing a wide range of observations from multiple domains.

WESCML-SF1 contains four main classes: WESC_MonitoringLocation, WESC_Commodity, WESC_Agent and WESC_Procedure. WESC_MonitoringLocation is a specialisation of the O&M SF_SpatialSamplingFeature to describe the location associated with a water or energy supply or consumption monitoring devices. This may be a single point location or an area, such as an Australian Bureau of Statistics or local government region, where data from an aggregation of meters is reported. The WESC_MonitoringLocation is the feature that contains the ‘volume’ property being observed. The type of observation is considered to be an OM_TimeSeriesObservation, a specialized observation in which the result is a time-instant coverage that samples volumes at different times. WESC_MonitoringLocation properties allow describing the monitoring type (e.g. individual meter, smart meter, or locality), the location name (e.g. street address, region, jurisdiction.) and reference description.

The WESC_Commodity identifies the type of commodity (e.g. water, gas, electricity etc.) being sampled at the monitoring location, and the source of the commodity (e.g. desalination, stormwater, surface water, groundwater, solar, wind, grid, etc.). The WESC_Agent is a class to specify the name and type of utility, organisation, individual, population, or environment supplying or consuming the commodity. A relationship between the WESC_Agent and WESC_Commodity classes allows specifying the roles (consumer, supplier) each plays. The WESC_Procedure is a specialization of O&M ObservationProcess to specify the procedures used to determine and/or aggregate water and energy observations. It includes the number of meters, the number of readings and the statistical method used to generate the results.

In addition to the four feature classes, three code lists are included to specify the content for specific properties. These are terms specifying the types of agents (e.g. residential, commercial, industrial, agricultural, municipal), the commodity source (e.g. desalination, groundwater, hydroelectricity, coal fired power stations, etc.) and statistical methods (e.g. average, interpolation, median, modelled).

WESC Vocabularies

Controlled vocabularies for WESC

A key component of WESC is the set of agreed and shared vocabularies used to encode the content for each XML instance, such as result method used to measure a meter reading and units of measure.

WESC Tools

Tools have been developed to support use of WESCML and to help data custodians deploy data infrastructure configured for WESCML.

docker-geoserver allows data custodians to create a deployment of Geoserver using Docker containerization technologies.

wesc-sf0-node extends docker-geoserver with configurations of the WESCML-SF0 data model so that data custodians can plug in their WESC datasets and deliver them via WESCML and standardised OGC web services at a push of a button thus speeding up and easing the process of data provision.

About

The Water and Energy Supply and Consumption (WESC) data standard has been developed by CSIRO with partners to support a common information model for supply and consumption of water and energy, and its delivery via standardised data services. The WESC information model allows for data to be delivered in a consistent format regardless of how it is represented as a data format, though there are XML data formats developed and hosted on Github.

WESCML is currently used in 2 data projects in Australia. In the AURIN project, the WESCML is being used to standardise water/energy supply/consumption data from across 16 different water and energy companies. The WESC information model is being used in the National Energy Analytics and Research (NEAR) Program for developing standardised energy data views at a national scale, e.g. zone substation electricity supply.
